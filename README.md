# Imperial College London - Machine Learning and Artificial Intelligence - Black Box Optimisation Capstone Project

Given how Machine Learning (ML) and Artificial Intelligence (AI) are fast becoming ubiquitous in all walks of life, completing a Professional Certificate in ML & AI, which this black-box optimisation capstone project is a part of, was my way to discover what exactly ML and AI are, as well as a constructive and fun way to learn Python.

## PROJECT OVERVIEW

This Black-Box Optimisation (BBO) project replicates real-world engineering and machine learning scenarios where a multi-dimensional "black-box" function needs to be optimised. Here, the optimisation goal is maximisation over eight independent BBO functions with hidden features such as non-linearity, multiple local maxima and noise. An initial, limited sample dataset is provided for each BBO function. Additionally, each BBO function may be queried once per week, up to a maximum of 13 query submissions. These constraints replicate real-world scenarios where function evaluations can be monetarily costly and/or time-consuming, either of which leads to a limited evaluation budget. E.g. the tuning of a complex model's hyperparameters, or devising optimal parameters for industrial manufacturing or chemical processes.

As mentioned, the objective is to maximise eight BBO functions. However, this isn't straightforward, due to:

* Sparse initial data vs input feature dimensionality.
* The limited available additional function evaluations (13 in total per function).
* Six-decimal-place granularity for each input feature dimension.

## DATA

* All inputs are numeric spanning the unit range [0, 1).

## MODEL

The model performs Bayesian Optimisation via a Gaussian Process Regressor (GPR) surrogate. Multiple anisotropic kernels (e.g. Matern 1/2, Matern 3/2, Matern 5/2, Radial Basis Function (RBF) and Rational Quadratic), x input transform, and y output transform combinations are trialled to generate Leave-One-Out Cross-Validation (LOOCV) Standardised Mean Square Error (SMSE) scores. The model automatically selects the combination with the best (i.e. lowest) y-original space SMSE score to perform Upper Confidence Bound (UCB) and Expected Improvement (EI) acquisition function maximisation over. A range of UCB _beta_ and EI _xi_ parameter values are used to provide a selection of potential next query submission points depending upon exploration/exploitation preference. This model ultimately allows users to methodically select black-box optimisation evaluation points in a principled manner.

LOOCV is calculated using an analytical formula that only requires the GPR to be fit once using the complete dataset; this is instead of once per data sample being "left out". This provides a huge performance boost since with only a single fit, the GPR kernel's parameters only need to be optimised once, similarly.

The SMSE score was originally calculated from the results of predicting every available y output. However, from query submission 11 onwards, an additional _"SMSE_Top"_ score was calculated based on the prediction results for just the top quarter of y outputs. This metric only cares about highest y outputs and helps focus on the maximisation task at hand. (To allow for SMSE and SMSE_Top to be directly comparable, when converting Mean Square Error (MSE) into SMSE, both use the same variance calculated from all y outputs, var(y), in the denominator. I.e. SMSE_Top does not use the variance calculated from just the top quarter of y outputs.)

Given that y output transforms are used, where GPR model _mean_ predictions are in y-transformed space, Monte Carlo is employed to inverse transform such _mean_ predictions (along with the model's outputted uncertainty, _std_) back into y-original space. Monte Carlo is used since some transforms are non-linear.

## HYPERPARAMETER OPTIMISATION

The model tunes i.) GPR kernel, ii.) x input transform, and iii.) y output transform using exhaustive grid search.

### GPR kernels:

| Basic   Kernel Names              | Kernel   Composition                                          |
|-----------------------------------|---------------------------------------------------------------|
| Matern 2.5                        | ConstantKernel() *   Matern(nu=2.5)                           |
| Matern 1.5                        | ConstantKernel() *   Matern(nu=1.5)                           |
| Matern 0.5                        | ConstantKernel() *   Matern(nu=0.5)                           |
| RBF                               | ConstantKernel() *   RBF()                                    |
| Rational Quad                     | ConstantKernel() *   RationalQuadratic()                      |
| Periodic                          | ConstantKernel() *   ExpSineSquared()                         |
| Locally   Periodic                | ConstantKernel() *   RBF() * ExpSineSquared()                 |
| Linear + RBF                      | DotProduct() +   ConstantKernel() * RBF()                     |
| Matern 2.5   +  WhiteKernel       | ConstantKernel() *   Matern(nu=2.5) + WhiteKernel()           |
| Matern 1.5   +  WhiteKernel       | ConstantKernel() *   Matern(nu=1.5) + WhiteKernel()           |
| Matern 0.5   +  WhiteKernel       | ConstantKernel() *   Matern(nu=0.5) + WhiteKernel()           |
| RBF +  WhiteKernel                | ConstantKernel() *   RBF() + WhiteKernel()                    |
| Rational Quad   +  WhiteKernel    | ConstantKernel() *   RationalQuadratic() + WhiteKernel()      |
| Periodic   +  WhiteKernel         | ConstantKernel() *   ExpSineSquared() + WhiteKernel()         |
| Locally   Periodic +  WhiteKernel | ConstantKernel() *   RBF() * ExpSineSquared() + WhiteKernel() |
| Linear + RBF   +  WhiteKernel     | DotProduct() +   ConstantKernel() * RBF() + WhiteKernel()     |

The above are basic kernels with and without an additive WhiteKernel.

| Complex   Kernel Names                           | Kernel   Composition                                                                                     |
|--------------------------------------------------|----------------------------------------------------------------------------------------------------------|
| i). Local Sens + Mat 2.5 +  WhiteKernel          |  ConstantKernel() * Matern(nu=1.5) +   ConstantKernel() * Matern(nu=2.5) + WhiteKernel()                 |
| ii.) Ridge + RQ +  WhiteKernel                   |  DotProduct() + ConstantKernel() *   RationalQuadratic() + WhiteKernel()                                 |
| iii.) Local Sens + Ridge + Mat 2.5 + WhiteKernel | ConstantKernel() *   Matern(nu=1.5) + DotProduct() + ConstantKernel() * Matern(nu=2.5) +   WhiteKernel() |

i.) Combining a global smooth trend with a high-frequency local component.

ii.) Global linear trend overlaid with multi-scale non-linear fluctuations.

iii.) Combining a global trend with local peaks and a ridge.

| Name            | Description                                                               |
